# SPEC: Cheat Engine 式扫描工作流

## 概述

将 Bit Slicer 的搜索体验改造为类似 Cheat Engine 的"首次扫描 → 再次扫描 → 撤销扫描"工作流，让用户清晰地知道当前处于扫描的哪个阶段，并支持多级撤销。

## 设计原则

1. **底层搜索引擎零改动** — `ZGSearchForData`、`ZGNarrowSearchForData` 等函数完全不变
2. **增量式 UI 改动** — 在现有工具栏上增加按钮，不重构整体布局
3. **向后兼容** — 现有的搜索框、表达式、存储值 token 等功能保持不变
4. **符合 macOS 设计语言** — 按钮样式与现有工具栏一致

## 用户工作流

### 首次扫描（New Scan）

1. 用户选择目标进程
2. 选择数据类型（int32、float、string 等）
3. 在搜索框输入搜索值（如 `100`）
4. 选择搜索条件（等于、大于、小于等）
5. 点击 **「新扫描」** 按钮（或按 Enter）
6. 系统扫描目标进程的全部内存
7. 结果显示在变量表中，状态栏显示「找到 N 个结果，第 1 轮」

### 再次扫描（Next Scan）

1. 用户在游戏中改变值（如生命值从 100 变成 80）
2. 在搜索框输入新值（如 `80`）
3. 点击 **「再次扫描」** 按钮
4. 系统 **只扫描上一次结果中的地址**（不重新扫描全内存）
5. 结果更新，状态栏显示「找到 N 个结果，第 2 轮」

### 撤销扫描（Undo Scan）

1. 用户误操作或想回退到上一轮结果
2. 点击 **「撤销扫描」** 按钮
3. 系统恢复到上一轮的变量列表和搜索结果
4. 状态栏轮次减 1
5. 可多次撤销，直到回到第 1 轮

## UI 变化

### 工具栏布局

```
现有:
[Target ▾] [Type ▾] [Search ▾] [Op ▾] <---flex---> [📦Store] [🔍 Search Field]

改为:
[Target ▾] [Type ▾] [Search ▾] [Op ▾] <---flex---> [🔍New] [🔄Re] [↩Undo] [📦Store] [🔍 Search Field]
```

- **「新扫描」** 🔍 — SF Symbol `magnifyingglass`，始终可用
- **「再次扫描」** 🔄 — SF Symbol `arrow.clockwise`，仅在有上一轮结果时可用
- **「撤销扫描」** ↩ — SF Symbol `arrow.uturn.backward`，仅在有扫描历史时可用

### 状态栏

在现有状态栏基础上增加轮次指示：

```
找到 1,234 个值  |  第 3 轮
```

### 按钮行为映射

| 用户操作 | 等效现有行为 | 搜索引擎调用 |
|---------|-------------|-------------|
| 点击「新扫描」 | 清空变量表 + 搜索 | `ZGSearchForData` |
| 点击「再次扫描」 | 基于现有变量搜索 | `ZGNarrowSearchForData` |
| 点击「撤销扫描」 | 恢复旧 variables + searchResults | 无搜索，纯状态恢复 |
| 在搜索框按 Enter | 取决于当前状态（见下文） | 自动选择 |

### Enter 键智能行为

当用户在搜索框按 Enter 时：
- **无历史结果**（第 0 轮）→ 等效「新扫描」
- **有历史结果**（第 1+ 轮）→ 等效「再次扫描」

这保持了现有用户的肌肉记忆。

## 搜索状态机

```
         ┌──────────┐
         │   idle   │ ← 初始状态 / 进程切换 / 数据类型切换
         └────┬─────┘
              │ 用户点击「新扫描」或 Enter
              ▼
         ┌──────────┐
         │ 第 1 轮  │ ← _searchResults 有结果
         └────┬─────┘
              │ 用户点击「再次扫描」或 Enter
              ▼
         ┌──────────┐
         │ 第 2 轮  │ ← _searchResults 被 narrow
         └────┬─────┘
              │ 用户点击「撤销扫描」
              ▼
         ┌──────────┐
         │ 第 1 轮  │ ← 恢复到上一轮
         └──────────┘
```

**状态重置条件**（回到 idle）：
- 切换目标进程
- 切换数据类型
- 用户显式点击「新扫描」
- 进程退出/重启

## 边界情况

1. **搜索框为空时** — 「新扫描」和「再次扫描」禁用
2. **搜索框按 Enter 但无值** — 什么都不做
3. **搜索框输入表达式**（如 `58 * 8`）— 正常解析，与现有行为一致
4. **使用存储值 token**（`$variable`）— 正常工作，新扫描/再次扫描都支持
5. **搜索被取消** — 不计入轮次，保持当前状态
6. **搜索无结果** — 计入轮次，变量表清空，但可以撤销回退
7. **进程退出** — 清空历史，回到 idle
8. **多级撤销到第 1 轮后再撤销** — 按钮灰掉，无法继续撤销
9. **已有变量手动添加** — 这些变量在再次扫描时保持不变（现有行为）

## 本地化

新增字符串需要支持 3 种现有语言（+ 可选简中）：
- `en.lproj/Localizable.strings` — English
- `es.lproj/Localizable.strings` — Español
- `ru.lproj/Localizable.strings` — Русский
- `zh.lproj/Localizable.strings` — 简体中文（可选，目前目录不存在）

新增键：
- `newScanButton` — "New Scan" / "新扫描"
- `nextScanButton` — "Next Scan" / "再次扫描"
- `undoScanButton` — "Undo Scan" / "撤销扫描"
- `scanRoundFormat` — "Round %lu" / "第 %lu 轮"
